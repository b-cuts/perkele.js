npm

  • https://npmjs.org/
  • by @izs
  • From FAQ: Contrary to the belief of many, "npm" is not in fact an abbreviation for "Node Package Manager". It is a recursive bacronymic abbreviation for "npm is not an acronym".
  • js lib dependency management and distribution
  • (mostly) for Node.js
  • distributed with Node.js
  • 22 063 packages (2013-02-01)

npm

  • package.json
  • npm install -g [package] for command line tools
  • npm install [package] for local packages in node_modules/
  • packages have their local node_modules dir for dependencies
  • npm + Grunt == WORKFLOW_AWESOME
  • npm install && grunt and you're all set up!

Bower

  • http://twitter.github.com/bower/
  • by Twitter
  • npm install -g bower
  • images, CSS, JavaScript
  • offers only low-level dependency management
  • 869 components (2013-02-01)
  • used by Yeoman
  • flat components dir with all dependencies

Bower

  • bower install jquery creates 
    components/
└── jquery
    ├── component.json
    ├── composer.json
    ├── jquery.js
    └── jquery.min.js

Bower

  • bower install jquery
  • bower install git://github.com/components/jquery.git
  • bower install http://foo.com/jquery.awesome-plugin.js
  • bower install jquery#1.8.1
  • bower install --save jquery adds entry to component.json
  • bower update jquery-ui

Bower

  • component.json:
    {
  "name": "myProject",
  "version": "1.0.0",
  "main": ["./path/to/app.css", "./path/to/app.js", "./path/to/sprite.img"],
  "dependencies": {
    "jquery": "~1.7.2"
  }
}

Ender

  • http://ender.jit.su/
  • npm install -g ender
  • dependency manager and build tool
  • CommonJS modules
  • uses npm modules that have the ender keyword in their package.json
  • 212 registered packages (2013-02-01)

Ender

  • ender build backbone creates 
    .
├── ender.js
├── ender.min.js
└── node_modules
    ├── backbone
    └── ender-js
  • ender add jquery@1.8
    .
├── ender.js
├── ender.min.js
└── node_modules
    ├── backbone
    ├── ender-js
    └── jquery
  • demo: ender info

volo

  • http://volojs.org/
  • by @jrburke
  • npm install -g volo
  • AMD or CommonJS modules
  • dependency manager and project creation tool that favors GitHub for the package repository
  • volo key in package.json
  • automating workflow with a volofile
  • kinda like Grunt + Bower, with AMD focus

volo

  • volo create volo-test
    .
├── package.json
├── README.md
├── tools
│   ├── build.js
│   └── r.js
├── volofile
└── www
    ├── index.html
    └── js
        ├── app
        │   └── main.js
        ├── app.js
        └── lib
            └── require.js

volo

  • volo search underscore
  • volo add underscore
  • also volo add -amd documentcloud/underscore exports=_
  • volo build

component

  • https://github.com/component/component
  • http://component.io/
  • by @tjholowaychuk (visionmedia), know this guy!
  • CommonJS modules
  • the original component.json
  • incompatible with Bower's component.json
  • components can include styles, images, scripts, etc.
  • uses GitHub repos
  • dependencies in component.json

component

  • component create component-test
    component-test/
├── component.json
├── component-test.css
├── History.md
├── index.js
├── Makefile
├── Readme.md
└── template.html

    "template.js"

  ]
}

Jam

  • http://jamjs.org/
  • npm install -g jamjs
  • browser first
  • AMD modules
  • deps and config in the jam key in package.json
  • 400 published packages (2013-02-22)

Jam

  • jam install underscore
    .
└── jam
    ├── require.config.js
    ├── require.js
    └── underscore
        ├── package.json
        └── underscore.js
  • also ls, upgrade, compile, publish etc.
  • also .jamrc for e.g. repo config
